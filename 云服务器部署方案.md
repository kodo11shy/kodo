# 托班管理系统 · 云服务器部署方案

> 服务器：腾讯云轻量 4核4G Ubuntu
> 后端：FastAPI + SQLAlchemy + PostgreSQL
> 反向代理：Nginx + HTTPS
> 小程序要求：生产环境必须使用 HTTPS 域名，不能直接使用 IP 或 localhost

## 一、推荐路线

第一次上线建议用“手动部署”：

```text
1. 服务器安装 Python / PostgreSQL / Nginx
2. 上传 backend 代码
3. 配置 .env
4. systemd 托管 FastAPI
5. Nginx 反向代理 /api 和 /uploads
6. 配置 HTTPS
7. 微信公众平台配置 request/uploadFile 合法域名
```

Docker 和 1Panel 可以后续再做，不建议第一次上线时同时引入太多变量。

## 二、服务器初始化

```bash
sudo apt update
sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv postgresql postgresql-contrib nginx
sudo systemctl enable --now postgresql
sudo systemctl enable --now nginx
```

## 三、创建数据库

请把下面的 `请换成强密码` 换成真实强密码。

```bash
sudo -u postgres psql
```

进入 psql 后执行：

```sql
CREATE USER tuoban WITH PASSWORD '请换成强密码';
CREATE DATABASE tuoban_db OWNER tuoban;
GRANT ALL PRIVILEGES ON DATABASE tuoban_db TO tuoban;
\q
```

验证：

```bash
psql "postgresql://tuoban:请换成强密码@127.0.0.1:5432/tuoban_db" -c "select 1;"
```

## 四、上传项目

建议目录：

```bash
sudo mkdir -p /opt/tuoban
sudo chown -R ubuntu:ubuntu /opt/tuoban
mkdir -p /opt/tuoban/uploads
```

上传或拉取代码后，目录应类似：

```text
/opt/tuoban/backend
/opt/tuoban/uploads
```

如果用 `scp`：

```bash
scp -i CC.pem -r backend ubuntu@服务器IP:/opt/tuoban/
```

如果用 Git：

```bash
cd /opt/tuoban
git clone 你的仓库地址 repo
cp -r repo/backend ./backend
```

## 五、安装后端依赖

当前项目依赖以 `backend/requirements.txt` 为准，不要手写散装依赖。

```bash
cd /opt/tuoban/backend
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

## 六、配置后端环境变量

创建：

```bash
nano /opt/tuoban/backend/.env
```

示例：

```env
APP_ENV=production
APP_NAME=托班管理系统 API
API_PREFIX=/api

DATABASE_URL=postgresql+psycopg://tuoban:请换成强密码@127.0.0.1:5432/tuoban_db
TOKEN_SECRET=请换成至少32位随机密钥
TOKEN_EXPIRE_HOURS=168

DEFAULT_TEACHER_NAME=管理员
DEFAULT_TEACHER_PASSWORD=请上线后立即更换

AUTO_CREATE_TABLES=true
UPLOAD_ROOT=/opt/tuoban/uploads

WECHAT_APP_ID=
WECHAT_APP_SECRET=
WECHAT_MOCK_LOGIN=false
```

说明：

- `DATABASE_URL` 当前项目使用 `psycopg`，不是 `psycopg2`。
- `UPLOAD_ROOT` 是当前代码读取的配置名，不是 `UPLOAD_DIR`。
- 第一次上线可保留 `AUTO_CREATE_TABLES=true` 自动建表；后续如果引入正式迁移工具，再改成 `false`。
- 正式微信登录需要填写 `WECHAT_APP_ID/WECHAT_APP_SECRET`，生产环境建议 `WECHAT_MOCK_LOGIN=false`。
- `.env` 不要提交到仓库。

## 七、本机验证后端

```bash
cd /opt/tuoban/backend
source .venv/bin/activate
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

另开一个 SSH 窗口验证：

```bash
curl http://127.0.0.1:8000/api/health
```

正常应返回：

```json
{
  "status": "ok",
  "service": "托班管理系统 API",
  "env": "production",
  "checks": {
    "database": {
      "ok": true,
      "message": "connected",
      "dialect": "postgresql"
    },
    "upload_dir": {
      "ok": true,
      "message": "/opt/tuoban/uploads"
    }
  }
}
```

注意：当前入口是 `app.main:app`，不是 `main:app`。

## 八、配置 systemd

创建服务：

```bash
sudo nano /etc/systemd/system/tuoban-backend.service
```

写入：

```ini
[Unit]
Description=Tuoban Backend API
After=network.target postgresql.service

[Service]
User=ubuntu
Group=ubuntu
WorkingDirectory=/opt/tuoban/backend
EnvironmentFile=/opt/tuoban/backend/.env
ExecStart=/opt/tuoban/backend/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tuoban-backend
sudo systemctl status tuoban-backend
```

查看日志：

```bash
sudo journalctl -u tuoban-backend -f
```

## 九、配置 Nginx

创建：

```bash
sudo nano /etc/nginx/sites-available/tuoban
```

没有域名前可先用 IP 测试 HTTP；正式小程序必须换成 HTTPS 域名。

```nginx
server {
    listen 80;
    server_name 你的域名.com;

    client_max_body_size 50M;

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        alias /opt/tuoban/uploads/;
        add_header Cache-Control "public, max-age=86400";
    }
}
```

启用：

```bash
sudo ln -s /etc/nginx/sites-available/tuoban /etc/nginx/sites-enabled/tuoban
sudo nginx -t
sudo systemctl reload nginx
```

验证：

```bash
curl http://你的域名.com/api/health
```

## 十、配置 HTTPS

微信小程序生产环境必须 HTTPS。

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名.com
```

验证：

```bash
curl https://你的域名.com/api/health
```

## 十一、小程序域名配置

上线前需要在微信公众平台配置合法域名。

推荐域名：

```text
request 合法域名：https://你的域名.com
uploadFile 合法域名：https://你的域名.com
downloadFile 合法域名：https://你的域名.com
```

小程序代码里也需要把本地地址：

```js
http://192.168.1.8:8001
```

切换成：

```js
https://你的域名.com
```

建议后续把本地/生产 API 地址抽成单独配置文件，避免每次手改 `app.js`。

## 十二、防火墙

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable
sudo ufw status
```

后端 8000 只监听 `127.0.0.1`，不需要对公网开放。

## 十三、更新发布

```bash
cd /opt/tuoban/backend
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart tuoban-backend
curl https://你的域名.com/api/health
```

如果使用 Git：

```bash
cd /opt/tuoban
git pull
sudo systemctl restart tuoban-backend
```

## 十四、备份

创建备份目录：

```bash
mkdir -p /opt/tuoban/backups
```

手动备份：

```bash
pg_dump "postgresql://tuoban:请换成强密码@127.0.0.1:5432/tuoban_db" > /opt/tuoban/backups/tuoban_$(date +%Y%m%d_%H%M%S).sql
tar -czf /opt/tuoban/backups/uploads_$(date +%Y%m%d_%H%M%S).tar.gz -C /opt/tuoban uploads
```

定时备份可放入 `crontab -e`：

```cron
0 3 * * * pg_dump "postgresql://tuoban:请换成强密码@127.0.0.1:5432/tuoban_db" > /opt/tuoban/backups/tuoban_$(date +\%Y\%m\%d).sql
0 4 * * * find /opt/tuoban/backups -name "*.sql" -mtime +30 -delete
```

## 十六、常见问题

### 1. systemd 启动失败

看日志：

```bash
sudo journalctl -u tuoban-backend -n 100 --no-pager
```

常见原因：

- `.env` 里的数据库密码不对；
- `ExecStart` 写成了 `main:app`；
- 依赖没装完整；
- 上传目录权限不足。

### 2. 小程序提示 url not in domain list

需要在微信公众平台配置合法域名，且必须是 HTTPS 域名。开发阶段可以在微信开发者工具勾选“不校验合法域名”，生产不能这样。

### 3. 图片 404

检查：

```bash
ls -lah /opt/tuoban/uploads
curl https://你的域名.com/uploads/某张图片路径
```

并确认 Nginx `location /uploads/` 的 `alias` 指向 `/opt/tuoban/uploads/`。
