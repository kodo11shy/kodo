## 当前主要问题

### 1. 演示照片记录存在，但真实图片文件缺失（已修复）

当前数据库：

- `photos` 表有 30 条记录；
- 但 `backend/uploads/` 里没有对应 jpg 文件；
- 日志中出现大量 `/uploads/photos/... 404 Not Found`。

影响：

- 首页精彩瞬间破图；
- 家长照片墙破图；
- 餐食照片/作业照片可能破图；
- 演示观感会明显扣分。

建议优先修：

- 让 `seed.py` 同时生成真实占位图片文件；
- 或把演示照片改为小程序内置图片资源；
- 或先不返回精选照片，避免破图。

2026-06-06 更新：

- `backend/seed.py` 已支持生成照片记录时同步创建真实 jpg 演示图；
- 已为当前本地开发库补齐缺失照片文件；
- 当前 `tuoban_dev.db` 照片记录缺失文件数为 0；
- `GET /uploads/photos/...jpg` 已验证返回 200。

### 2. 视觉只改了首页和全局基调

v2 方向已经定了，但很多页面还没真正应用：

- 家长首页
- 家长成长/作业/照片
- 老师工作台
- 老师学生/收费/餐食/通知/设置
- 各空状态

建议下一步让 Claude Code 集成 v2 图片和 UI 方案。

### 3. 小程序生产 API 地址还没配置

当前 `miniprogram/app.js` 仍是本地联调地址：

```js
http://192.168.1.8:8000/api
http://127.0.0.1:8000/api
```

上线前必须改成：

```js
https://api.your-domain.com/api
https://api.your-domain.com
```

并删除本地 fallback，避免审核/生产环境访问内网地址。

### 4. 部署文档需要更新

当前 `云服务器部署方案.md` 里有几处需要修正：

- systemd 启动命令应使用 `app.main:app`，不是 `main:app`；
- 生产后端建议只监听 `127.0.0.1:8000`，由 Nginx 对外暴露 HTTPS；
- Python 依赖应直接使用 `pip install -r requirements.txt`；
- 当前项目使用 `psycopg[binary]`，不是 `psycopg2-binary`；
- `/api/health` 暂未实现，验证路径应使用 `/` 或 `/api/public/homepage`。

### 5. 生产安全还没收口

上线前必须做：

- 修改默认老师密码；
- 设置强 `TOKEN_SECRET`；
- 禁止使用开发 SQLite；
- 使用 PostgreSQL；
- 设置 HTTPS；
- 配置微信合法域名；
- 备份数据库和 uploads；
- 不把密钥放进小程序前端。

## 云服务器建议配置

已有腾讯云轻量 4核4G Ubuntu，可以支撑一期。

### 推荐生产架构

```text
微信小程序
  ↓ HTTPS
Nginx / OpenResty
  ├─ /api/     → FastAPI 127.0.0.1:8000
  └─ /uploads/ → /home/ubuntu/tuoban/uploads/

FastAPI
  ↓
PostgreSQL
```

### 服务器软件

- Ubuntu 22.04 或 24.04
- Python 3.11+
- PostgreSQL 15/16
- Nginx 或 OpenResty
- Certbot / acme.sh
- UFW 防火墙
- fail2ban 可选

### 后端生产 `.env`

```env
APP_NAME=托班管理系统 API
APP_ENV=production
API_PREFIX=/api
DATABASE_URL=postgresql+psycopg://tuoban:强密码@127.0.0.1:5432/tuoban
TOKEN_SECRET=至少32位随机字符串
TOKEN_EXPIRE_HOURS=168
DEFAULT_TEACHER_NAME=管理员
DEFAULT_TEACHER_PASSWORD=上线前改强密码
AUTO_CREATE_TABLES=true
UPLOAD_ROOT=/home/ubuntu/tuoban/uploads
```

说明：

- 首次部署可以 `AUTO_CREATE_TABLES=true`；
- 后续最好改为迁移脚本/Alembic 管理表结构；
- 不要在小程序里放数据库密码、AppSecret、AI Key。

### systemd 建议

```ini
[Unit]
Description=Tuoban Backend API
After=network.target postgresql.service

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/tuoban/backend
EnvironmentFile=/home/ubuntu/tuoban/backend/.env
ExecStart=/home/ubuntu/tuoban/backend/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

### Nginx 建议

```nginx
server {
    listen 80;
    server_name api.your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.your-domain.com;

    client_max_body_size 50M;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        alias /home/ubuntu/tuoban/uploads/;
        access_log off;
        expires 7d;
    }
}
```

### 微信小程序后台需要配置

生产环境需要在小程序后台配置：

- request 合法域名：`https://api.your-domain.com`
- uploadFile 合法域名：`https://api.your-domain.com`
- downloadFile 合法域名：`https://api.your-domain.com`
- web-view 业务域名：如果 H5 报告使用 `https://h5.your-domain.com`，则配置该域名

注意：

- 正式版必须 HTTPS；
- 本地“开发工具不校验合法域名”只适合开发；
- 建议使用域名，不建议生产直接用 IP；
- 国内云服务器 + 域名通常还需要完成备案。

## 智能体配置建议

智能体现在还没有真正实现，不建议马上接大模型。

### 一期：纯代码查询智能体

目标：只查数据库，不编造。

能力范围：

- “张小明今天签到了吗？”
- “今天吃了什么？”
- “这周作业完成了吗？”
- “本月收费缴了吗？”

实现方式：

- 后端新增 `agent` 模块；
- 使用固定意图识别：关键词/规则/白名单；
- 每个问题映射到明确 SQL/API；
- 查不到就回答“暂无记录”，不自由发挥；
- 所有问答写日志。

建议环境变量：

```env
AGENT_ENABLED=false
AGENT_MODE=rule_based
AGENT_MAX_LOOKBACK_DAYS=30
```

### 企业微信接入

如果要接企业微信，需要准备：

```env
WEWORK_CORP_ID=
WEWORK_AGENT_ID=
WEWORK_SECRET=
WEWORK_TOKEN=
WEWORK_AES_KEY=
```

并新增：

- 企业微信回调接口；
- 消息验签；
- 家长/群聊/学生绑定关系；
- 权限校验；
- 敏感数据过滤。

### 二期：接入大模型

等一期规则智能体稳定后再接。

建议：

- API Key 只放后端 `.env`；
- 不让模型直接访问数据库；
- 由后端先查结构化数据，再把“已查到的数据”交给模型润色；
- 系统提示词要求：不知道就说不知道，没有记录就说暂无记录；
- 对健康、收费、隐私类内容做权限限制。

可选环境变量：

```env
LLM_ENABLED=false
LLM_PROVIDER=openai_or_claude
LLM_API_KEY=
LLM_MODEL=
```
