# 🏫 托班管理系统 · 从代码到上线的完整路线图

> **目标**：把本地开发的小程序后端部署到腾讯云服务器，前端通过微信小程序上线，Hermes打通数据自动生成月度报告海报
> **当前状态**：后端代码已写好（FastAPI + SQLite），小程序前端已写好，代码已通过Syncthing同步到云服务器
> **阅读对象**：开发者（Claude Code / Codex / 人类）

---

## 一、项目全景

```
老师在小程序操作
    │
    ▼
小程序前端（微信托管，免费）
    │  HTTPS 请求
    ▼
Nginx 反向代理（云服务器）
    │
    ▼
FastAPI 后端（云服务器 :8000）
    │
    ├── PostgreSQL 数据库（存学生、签到、作业、照片...）
    ├── 文件存储（照片原图/缩略图）
    │
    ▼
Hermes（同一台服务器）
    ├── 读数据库 → 生成月度报告文案
    ├── 调海报API → 生成图片
    └── 发给你的微信
```

## 三、阶段详解

### 阶段 ① 服务器准备

#### 3.1 安装基础软件

```bash
# PostgreSQL
sudo apt update
sudo apt install -y postgresql postgresql-contrib
sudo systemctl enable --now postgresql

# Nginx
sudo apt install -y nginx
sudo systemctl enable --now nginx

# Python（确认已有 python3.10+）
python3 --version
```

#### 3.2 创建数据库

```bash
sudo -u postgres psql
```

在 psql 中执行：

```sql
CREATE USER tuoban WITH PASSWORD '设一个强密码，记下来！';
CREATE DATABASE tuoban_db OWNER tuoban;
GRANT ALL PRIVILEGES ON DATABASE tuoban_db TO tuoban;
\q
```

验证：

```bash
psql "postgresql://tuoban:设的密码@127.0.0.1:5432/tuoban_db" -c "SELECT 1;"
```

#### 3.3 创建项目目录

```bash
sudo mkdir -p /opt/tuoban/uploads
sudo chown -R ubuntu:ubuntu /opt/tuoban
```

### 阶段 ② 后端部署

#### 4.1 部署代码

> 代码已经在云服务器上了（通过 Syncthing 同步），路径在 Obsidian vault 里。

```bash
# 从 vault 复制到正式部署目录
cp -r /home/记事/ObsidianVault/石不语札记/01\ -\ 工作/托班管理系统/backend /opt/tuoban/
```

#### 4.2 安装 Python 依赖

```bash
cd /opt/tuoban/backend
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

#### 4.3 修改 .env 为生产配置

创建 `/opt/tuoban/backend/.env`：

```env
APP_ENV=production
APP_NAME=托班管理系统 API
API_PREFIX=/api

DATABASE_URL=postgresql+psycopg://tuoban:设的密码@127.0.0.1:5432/tuoban_db
TOKEN_SECRET=生成一个32位随机字符串（用 openssl rand -hex 32）
TOKEN_EXPIRE_HOURS=168

DEFAULT_TEACHER_NAME=管理员
DEFAULT_TEACHER_PASSWORD=上线后第一时间修改这个密码！

AUTO_CREATE_TABLES=true
UPLOAD_ROOT=/opt/tuoban/uploads
```

> **注意**：当前代码用的是 `psycopg`（v3），不是 `psycopg2`，`DATABASE_URL` 里的驱动名要写对。

#### 4.4 验证后端启动

```bash
cd /opt/tuoban/backend
source .venv/bin/activate
python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
```

另开终端测试：

```bash
curl http://127.0.0.1:8000/api/health
```

应返回包含 database ok 的 JSON。

#### 4.5 配置 systemd 服务（开机自启）

创建 `/etc/systemd/system/tuoban-backend.service`：

```ini
[Unit]
Description=Tuoban Backend API
After=network.target postgresql.service

[Service]
User=ubuntu
Group=ubuntu
WorkingDirectory=/opt/tuoban/backend
EnvironmentFile=/opt/tuoban/backend/.env
ExecStart=/opt/tuoban/backend/.venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tuoban-backend
sudo systemctl status tuoban-backend
```

查看日志：

```bash
sudo journalctl -u tuoban-backend -f
```

### 阶段 ③ 域名 + HTTPS（小程序必须）

#### 5.1 域名方案

小程序生产环境**要求 HTTPS 域名**，不能直接用 IP。可选方案：

| 方案 | 成本 | 说明 |
|------|------|------|
| **买新域名** | 约 30-80 元/年 | 推荐 `.xyz` 或 `.top`，最便宜 |
| **找回旧域名** | 0 元 | 之前的 `xxx.online.online`，如果还在 |
| **微信云开发** | 有免费额度 | 不买域名直接用微信提供的环境，但数据不能跟你现有服务器打通 |

> **建议**：花几十块买个新域名，一劳永逸。推荐在腾讯云 DNSPod 买，管理方便。

#### 5.2 DNS 解析

把域名解析到云服务器公网 IP：

```
云服务器公网IP查询：
curl ifconfig.me
```

在 DNSPod（腾讯云域名控制台）添加 A 记录：

```
类型: A
主机记录: @ 或 api
记录值: 【服务器公网IP】
```

#### 5.3 Nginx 配置

创建 `/etc/nginx/sites-available/tuoban`：

```nginx
server {
    listen 80;
    server_name 你的域名.com;

    client_max_body_size 50M;

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        alias /opt/tuoban/uploads/;
        add_header Cache-Control "public, max-age=86400";
    }
}
```

启用：

```bash
sudo ln -s /etc/nginx/sites-available/tuoban /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

#### 5.4 HTTPS 证书（Let's Encrypt，免费）

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d 你的域名.com
```

证书到期前会自动续期，不用管。

#### 5.5 防火墙

```bash
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw --force enable
sudo ufw status
```

#### 5.6 验证

```bash
curl https://你的域名.com/api/health
```

### 阶段 ④ 小程序上线

#### 6.1 注册认证小程序

1. 打开 [mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 注册小程序账号（用营业执照或个体工商户）
3. 支付 **300 元认证费**（每年一次）
4. 等待审核（通常 1-3 个工作日）

> 认证通过后，你才算有了正式的「小程序身份」，才能配置合法域名和发布。

#### 6.2 配置合法域名

在小程序管理后台 →「开发」→「开发设置」→「服务器域名」：

```
request 合法域名: https://你的域名.com
uploadFile 合法域名: https://你的域名.com
downloadFile 合法域名: https://你的域名.com
```

#### 6.3 修改小程序前端配置

修改 `miniprogram/app.js` 中的 `apiBase`：

```js
// 原来（指向本地开发）
apiBase: 'http://192.168.1.8:8001/api'

// 改为（指向云服务器生产环境）
apiBase: 'https://你的域名.com/api'
apiOrigin: 'https://你的域名.com'
```

> **建议**：后续可以把 `apiBase` 抽出来放在一个 `config.js` 文件里，开发/生产切换更方便，不用每次都改 `app.js`。

#### 6.4 开发者工具测试

在微信开发者工具中：

1. 用正式小程序 AppID（认证后就有）
2. **去掉**「不校验合法域名」的勾
3. 测试登录、拍照上传、作业记录等全部流程

#### 6.5 提交审核发布

1. 开发者工具 →「上传」
2. 微信公众平台 →「版本管理」→「提交审核」
3. 审核通过后发布（审核通常 1-7 天）

### 阶段 ⑤ Hermes 数据打通 + 月度报告

这是你想要的核心功能——我在云端读数据库，自动生成月度报告海报发给家长。

#### 7.1 建立 Hermes 到数据库的连接

我（Hermes）跟后端跑在同一台服务器上，数据是通的。

```python
# 连接 PostgreSQL
import psycopg

conn = psycopg.connect("postgresql://tuoban:密码@127.0.0.1:5432/tuoban_db")
# 直接读库，不需要经过 API，内网直连
```

#### 7.2 月度报告内容结构

从数据库里取出一个学生上个月的全部数据：

| 数据来源 | 内容示例 | 数据库表 |
|---------|---------|---------|
| 出勤 | 本月出勤 22 天，全勤 | `attendance_records` |
| 作业 | 语文全对 15 次，数学全对 12 次 | `homework_records` |
| 评语 | 老师写了 8 条评语 | `teacher_remarks` |
| 照片 | 本月累计 36 张照片 | `photo_students` + `photos` |
| 餐食 | 特别记录 3 条 | `meal_student_notes` |
| 里程碑 | 记录了 2 个成长瞬间 | `growth_milestones` |

#### 7.3 报告生成流程

```
① 每月1号自动触发（cron 定时任务）
   ↓
② Hermes 查询数据库 → 生成学生月度报告文案
   ↓
③ 调用海报 API 生成精美图片
   ↓
④ 通过微信发送给你审核，或直接发给家长
```

> **数据获取方式**：初期可以直接读数据库。后续业务复杂了（权限校验、数据聚合），建议走后端 API——Hermes 也在内网，调自己 API 速度一样快，还能复用后端的权限和业务逻辑。

#### 7.4 海报 API 方案

生成图片（海报）的方案对比：

| 方案 | 费用 | 质量 | 说明 |
|------|------|------|------|
| **Python Pillow 手工拼** | 免费 | 中等 | 用代码画图，模板是固定的，调起来比较累 |
| **通义千问/文心一言图生图** | 按量计费 | 高 | 给文案让AI生图，风格统一，但文字可能不准 |
| **HTML → 截图（Puppeteer/Playwright）** | 免费 | 最高 | 先写 HTML 模板，用浏览器截图，排版最自由 |
| **Canva API 或稿定设计 API** | 有免费额度 | 高 | 有现成模板，但需要对接 |
| **调用我自己（视觉模型）** | 免费（已在用的模型） | 中高 | 我目前用的 MiMo 模型支持图文生成，可以试试 |

> **推荐路径**：先用 **HTML → 截图**方案（Playwright），写一个漂亮的海报 HTML 模板，再截图。效果最好，控制最细，完全免费，而且家长看到的是完整的图文排版。

#### 7.5 发送方式

```
你审核 → 你转发给家长（初期，安全可控）
或者
Hermes 自动发给家长（后期，需要家长微信绑定）
```

#### 7.6 数据库查询 SQL 参考（Hermes 用）

```sql
-- 某学生某月完整成长报告数据
SELECT * FROM (
  -- 作业
  SELECT hr.homework_date AS event_date, '作业' AS category,
         hr.subject || ' · ' || hr.accuracy_status AS title,
         hr.auto_comment AS description, hr.score
  FROM homework_records hr
  WHERE hr.student_id = $student_id
    AND hr.homework_date >= $month_start
    AND hr.homework_date < $month_end

  UNION ALL

  -- 评语
  SELECT tr.record_date, '评语' AS category,
         tr.mood_tag || ' · 老师评语' AS title,
         tr.content AS description, NULL
  FROM teacher_remarks tr
  WHERE tr.student_id = $student_id
    AND tr.record_date >= $month_start
    AND tr.record_date < $month_end

  UNION ALL

  -- 里程碑
  SELECT gm.milestone_date, '成长里程碑' AS category,
         gm.title, gm.description, NULL
  FROM growth_milestones gm
  WHERE gm.student_id = $student_id
    AND gm.milestone_date >= $month_start
    AND gm.milestone_date < $month_end
) AS timeline
ORDER BY event_date DESC;
```
