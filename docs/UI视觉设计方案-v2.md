# 托班小程序 UI 视觉设计方案 v2

日期：2026-06-06

## 设计关键词

安心、温暖、日常、陪伴、手作感。

这个小程序面对老师和家长，不应该像通用管理后台，也不应该像低幼玩具。视觉上要像一个干净、可信、有温度的托班空间：有自然光、植物、木质家具、画纸、书包、积木和孩子活动的生活感。

## 总体风格

- 图片：绘本感手绘插画，柔和水彩/蜡笔质感，避免硬矢量图标。
- 背景：浅暖底色，不用大面积强渐变。
- 卡片：白色或奶油白，轻阴影，边框极浅，圆角适中。
- 按钮：像“可触摸的软按钮”，颜色温和但清晰。
- 动效：轻微反馈即可，不做夸张弹跳。

## 色彩系统

主色：

- 家长端主色：`#FF8C5A`，暖橙，表达陪伴和亲近。
- 老师端主色：`#5B7FFF`，蓝紫，表达清晰、效率和可信赖。

辅助色：

- 成功/健康：`#36C2A0`
- 提醒/活动：`#FF7E5F`
- 温暖底色：`#FFF7EF`
- 老师端背景：`#F7F8FC`
- 家长端背景：`#FFFAF7`
- 正文深色：`#2F3A4C`
- 次级文字：`#7B8496`
- 分割线：`#ECEFF5`

## 首页

首页应该是“托班门面”，第一屏用 `banner-home-v2.png`。

建议：

- banner 全宽展示，圆角 `28rpx`，不要再用蓝色渐变 hero。
- 标题叠在右上留白处或 banner 下方，不要盖住孩子活动主体。
- 老师入口和家长入口做成两张温和入口卡，不使用粗重左边框。
- 收费、精彩瞬间、关于我们使用轻卡片，不要堆太多颜色。

## 老师端

老师端需要效率，但也不能冷冰冰。

建议：

- 背景：`#F7F8FC`
- 主按钮：蓝紫 `#5B7FFF`
- 工作台顶部可以用纯色蓝紫加轻浅纹理，不做重渐变。
- 九宫格图标可继续用 v1 生成的 `images/icons/icon-*.png`，但按钮卡片应更轻：白底、浅蓝小圆底、文字深色。
- 数据卡片强调扫描效率，少装饰，多留白。

## 家长端

家长端重点是“看到孩子被照顾得很好”。

建议：

- 背景：`#FFFAF7`
- 主按钮：暖橙 `#FF8C5A`
- 孩子卡片可以加入浅杏色背景和柔和头像圆形。
- 成长时间线用小贴纸感节点，不要像后台日志。
- 空状态统一用 v2 绘本图，不再只显示 emoji。

## 按钮规范

主按钮：

```css
background: #FF8C5A 或 #5B7FFF;
border-radius: 999rpx;
height: 88rpx;
box-shadow: 0 10rpx 24rpx rgba(..., 0.18);
font-size: 30rpx;
font-weight: 600;
```

次按钮：

```css
background: #FFFFFF;
border: 2rpx solid #E7EAF2;
color: #5B6476;
border-radius: 999rpx;
```

危险按钮不要大面积鲜红，使用浅红底 + 深红字，只有确认弹窗内再用红色实心。

## 卡片规范

```css
background: rgba(255,255,255,0.92);
border-radius: 24rpx;
box-shadow: 0 10rpx 30rpx rgba(49,64,90,0.08);
border: 1rpx solid rgba(255,255,255,0.8);
```

托班场景适合“轻卡片”，不适合厚重阴影和硬边框。

## 空状态规范

空状态不要只写“暂无数据”。

建议结构：

- 上方插画：220-260rpx
- 主文案：温和明确，例如“还没有照片”
- 次文案：解释下一步，例如“老师拍照并关联后，会出现在这里”
- 如有行动按钮，使用次按钮或小主按钮。

## 不建议继续使用

- 大面积蓝色渐变背景。
- emoji 作为主功能图标。
- 纯线框硬图标作为托班主视觉。
- 过冷的灰白管理后台风。
- 过度幼稚的糖果色和夸张卡通脸。

## 给 Claude Code 的集成建议

1. 先替换首页 banner 和 Logo。
2. 再替换四类空状态图。
3. 最后统一 `app.wxss` 的按钮、卡片、背景色。
4. 老师端和家长端分开设主色，不要全站一个蓝色。
5. 每改完一个页面，在微信开发者工具看 375x812 和较小屏，确认文字不压图、不溢出。
