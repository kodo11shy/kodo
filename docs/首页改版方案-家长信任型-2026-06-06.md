# 首页改版方案：家长信任型

日期：2026-06-06

## 结论

当前首页功能完整，但更像“系统入口页”。正式对外使用时，首页应该优先让家长产生信任，而不是优先展示老师和家长两个入口。

建议采用“家长信任型首页”：

1. 先展示托班氛围和安心感。
2. 再给家长快速联系入口。
3. 接着展示孩子日常和通知。
4. 最后补充收费、关于我们和老师入口。

## 当前首页的问题

### 入口优先级偏管理系统

老师入口和家长入口并列放在第一屏下方，会让页面像内部系统门户。对于第一次打开的家长，老师入口不是主要信息。

### 情绪价值还不够

托班首页需要传达“干净、温暖、可靠、孩子被照顾得很好”。当前页面有温暖底色和 banner，但模块顺序仍偏功能罗列。

### 精彩瞬间应该更前置

托班最有说服力的是孩子真实日常：吃饭、阅读、活动、作业、休息。当前“精彩瞬间”位置可以保留，但展示形式建议更像生活相册，而不是单纯轮播。

### 收费不应太像报价表

收费信息要透明，但不应该在视觉上像后台配置项。建议包装成“透明收费”模块，强调清楚、可咨询。

## 推荐首页结构

```text
1. 氛围大图
2. 机构名 + 安心陪伴语
3. 联系微信 / 拨打电话 / 家长入口
4. 精彩瞬间
5. 最新通知
6. 透明收费
7. 关于托班
8. 老师入口
```

## 三种可选方案对比

### 方案 A：家长信任型

这是当前最推荐方案。

适用场景：

- 小程序会对外发给新家长；
- 首页承担招生、咨询、了解托班的作用；
- 需要让用户第一眼觉得安心、温暖、可信。

优点：

- 转化路径清楚，微信和电话更容易被看到；
- 精彩瞬间前置，更容易建立信任；
- 老师入口弱化后，页面更像托班门面。

不足：

- 老师每天进入后台时会多一步寻找入口；
- 需要更认真处理图片和空状态，否则情绪表达会弱。

### 方案 B：成长日记型

适合家长已经绑定孩子后使用，不适合做默认对外首页。

结构示例：

```text
孩子今日状态
今日照片
餐食记录
作业反馈
成长点评
通知
联系老师
```

优点：

- 家长打开后能快速看到孩子今天发生了什么；
- 使用频率高，适合做家长端 Dashboard。

不足：

- 对新访客不友好；
- 未登录、未绑定孩子时内容会空；
- 不适合作为统一首页。

### 方案 C：功能入口型

这是当前首页更接近的方向。

结构示例：

```text
老师入口
家长入口
通知
照片
收费
关于我们
```

优点：

- 功能直接；
- 内测和开发联调方便；
- 老师、家长都能快速找到入口。

不足：

- 对外展示气质偏系统；
- 家长信任感建立不足；
- 老师入口过早出现，会抢占托班门面的第一印象。

### 推荐取舍

正式发布建议：

```text
默认首页：方案 A 家长信任型
家长登录后：首页/工作台采用方案 B 成长日记型
老师端：首页/工作台采用方案 C 管理入口型
```

这样三类页面各司其职：

- 对外首页负责建立信任和引导咨询；
- 家长端负责展示孩子日常；
- 老师端负责高效管理。

## 模块设计建议

### 1. 氛围大图

继续使用 `miniprogram/images/banner-home-v2.png`，但标题区域不要做成强卡片感。

建议：

- banner 保持大图第一视觉；
- 文案放在图片留白处或图片下方；
- 少用遮罩和厚阴影；
- 让图片承担“托班氛围”表达。

文案示例：

```text
星辰伴梦
放学后的安心陪伴
```

如果使用后台配置的欢迎语，可把欢迎语作为第二行说明。

### 2. 家长行动区

优先展示可操作按钮：

- 复制微信
- 拨打电话
- 家长入口

老师入口不建议和家长入口并列放大。可以放到底部，或以较小文本入口出现。

### 3. 精彩瞬间

建议由单纯轮播改成“1 大 + 2 小”的照片拼贴：

```text
大照片
小照片 小照片
```

如果后端只有 1 张照片，则单图展示；没有照片时使用 `empty-photos-v2.png`。

### 4. 最新通知

通知不建议放成强滚动条占据很高优先级。可以做成温和信息条：

```text
最新通知：五一放假安排
```

点击后仍弹窗查看详情。

### 5. 透明收费

收费模块标题建议：

```text
透明收费
```

展示前 2-3 项即可，长列表可以折叠。

视觉上建议用轻卡片，不要大号价格压过页面情绪。

### 6. 关于托班

关于我们模块不需要重复欢迎语太多。建议表达三类信息：

- 接送后照看
- 作业陪伴
- 餐食与成长记录

如果后续有更多配置项，可以从后台设置读取。

### 7. 老师入口

放在页面底部，样式弱化：

```text
老师登录
```

或放成右上角小文字入口。正式对外首页不建议把老师入口做成大卡片。

## 视觉原则

- 背景保持暖白，不要大面积强渐变。
- 入口按钮要清晰可点，少用 emoji。
- 卡片圆角控制在 `24rpx` 左右，阴影更轻。
- 字体层级收敛，首页不要出现太多大标题。
- 颜色以暖橙和自然白为主，蓝紫只给老师入口或后台区。
- 图像优先表达真实生活感，避免硬图标感。

## 可以保留的现有内容

- 联系方式接口和点击逻辑
- 通知弹窗逻辑


## 给 Claude Code 的协作建议

如果进入实现阶段，建议分两步：

1. 先调整 WXML 模块顺序和 CSS 视觉层级，不动接口。
2. 再根据照片数量优化“精彩瞬间”的拼贴展示。

实现完成后需要在微信开发者工具检查：

- 首页首屏是否自然展示机构名；
- 家长是否能在第一屏快速找到微信/电话；
- 没有照片、没有通知、没有收费配置时空状态是否正常；
- 小屏幕下按钮文字不挤压、不换行错乱；
- 老师入口是否仍然可找到。
