> **核心原则**：老师操作 ≤ 2步，绝不进入3级以上菜单

> **2026-06-05 更新**：取消"最近5个"推荐逻辑，改为直接展示全部学生网格，由老师自行选择。不依赖AI判断，老师最清楚是谁。

## 操作路径总结

| 场景 | 操作步骤 | 耗时 |
|------|----------|------|
| 拍作业本（有名字） | 拍照 → 自动识别 → 确认 ✓ | 1秒 |
| 拍作业本（无名字） | 拍照 → 点一下学生头像 | 2秒 |
| 拍单人活动照 | 拍照 → 点一下学生头像 | 2秒 |
| 拍合影 | 拍照 → 切换多人 → 勾选几个学生 | 3-4秒 |
| 忙时只管拍 | 连拍N张 → 下班花5分钟批量关联 | 分散 |

## 补充说明：OCR识别作业本名字的"傻瓜版"做法

如果你不想引入OCR技术，还有一个更简单粗暴的方案：

**方案：作业本封面贴二维码**

在为每个学生印制作业本封面的阶段，在名字旁边印一个二维码/条形码：
- 老师拍照时打开"扫一扫"模式
- 扫码自动识别学生身份
- 然后拍照，自动关联

这个方案技术难度最低，且几乎没有识别错误率。但前提是需要统一制作作业本。

**我建议的方案**：先用最简单的"拍照 → 点选学生"上线，后续再逐步加入OCR和二维码。功能能跑起来比一次做完更重要。
