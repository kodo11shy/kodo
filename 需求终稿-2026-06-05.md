# 🏫 托班管理系统 · 需求终稿

> **基于**：初版需求（2026-06-03）+ 与 Claude 完整讨论（2026-06-05）
> **状态**：全部模块已确认，具备开发条件

---

## 一、项目定位

| 项    | 内容                      |
| ---- | ----------------------- |
| 目标用户 | 托班老师（核心）、管理者、家长（只查看）    |
| 服务规模 | 约 20 名小学阶段儿童（语文+数学为主）   |
| 核心差异 | 做完→批改→讲解→改错，闭环辅导，不只是看着做 |
| 使用原则 | **系统给老师用的**，家长端零操作或极低操作 |
| 技术形态 | **微信小程序**，一个应用三种角色入口    |

---

## 二、小程序结构

```
打开小程序
        │
        ├─ 游客/访客 → 对外展示页
        │     ├─ 通知公告（首页循环滚动播放）
        │     ├─ 关于我们（环境·师资·理念）
        │     ├─ 收费标准（透明公开）
        │     └─ 精彩瞬间（照片/视频展示）
        │
        ├─ 老师入口 → 密码验证 → 工作台
        │     ├─ 拍照 📸  签到 ✅  作业 📋
        │     ├─ 成长档案 🌟  报告 📊  餐食 🍱
        │     ├─ 收费 💰  群聊 💬  设置 ⚙️
        │     └─ 招生营销 📢（对外展示管理）
        │
        └─ 家长入口 → 邀请码绑定微信 → 专属页面
              └─ 只能看到自己孩子的：
                    成长档案 · 作业记录 · 照片 · 老师评语 · 餐食记录
```

## 三、功能模块（全部已确认）

### 1️⃣ 学生档案管理 👶

| 层级 | 内容 | 填写时机 |
|------|------|----------|
| **第一组（必填）** | 姓名、性别、学校年级、家长联系方式、家庭地址 | 报名时 |
| **第二、三组（必填，需授权）** | 接送授权人、过敏史、既往病史、特殊注意事项 | 报名时，家长签告知书 |
| **第四组（选填）** | 家长2、紧急联系人、家庭情况 | 报名或后续 |
| **第五组（补充）** | 入学照、兴趣爱好、性格特点、薄弱科目、备注 | 老师日常完善 |

> **关联设计**：学生档案是所有模块的主数据，签到/作业/照片/餐食/评语均通过学生ID关联。
> **告知书**：涉敏感信息采集需家长签字确认，作为纠纷背书。

### 2️⃣ 签到签退 🏁

- 打开页面看到**全部学生网格**，分"已到/未到"区域
- **点一下名字 = 签到**，记录当前时间
- 签到时记录**谁送来的**（点完名字再点接送人）
- 顶部可切换 **🔵签到 / 🟠签退** 模式
- **长按名字 = 补签**，弹出对话框手动调时间 + 必填备注（标记为"补签"）
- 签退同理，已签退显示时间，未签退显示"在托"
- 首页顶部显示：今日在托人数 / 总人数

### 3️⃣ 拍照记录 📸

- 拍照后**展示全部学生网格**（不分页，20人4-5行），老师自己点选关联
- 支持**单人**（点一个）和**多人**（切换多选模式，合影用）
- 多选时**多对多关联**，照片只存一份
- 每步拍照支持**1~9张**（九宫格）
- 照片分类标签：作业照 / 活动照 / 餐食照 / 日常照
- 精选标记：好照片可标记为"精选"
- 语音输入：老师用**豆包输入法**完成语音转文字备注

### 4️⃣ 作业管理 📚 ← 核心差异化

**三步拍照闭环**（区别于其他托班只看着做）：

```
第1步：孩子做完交上来 → 拍"完成证明" → 标记"待批改"
第2步：老师批改打分 → 拍"批改情况" → 标记"已批改"
第3步：老师讲解 → 孩子改错 → 拍"改完成" → 标记"已完成"
```

| 字段 | 说明 |
|------|------|
| 科目 | 语文 / 数学（后续可扩展） |
| 三步照片 | 每步 1~9 张（九宫格） |
| 完成情况 | 全对 ✅ / 有错已讲解 ⚠️ / 待批改 ⏳ / 已完成 ✅ |
| 错题数 | 数字输入 |
| 语音备注 | 豆包输入法语音转文字 |
| 评分 | 1-10分滑动选择 |
| 自动评语 | 可选功能，根据数据自动生成文案，老师可改 |

> **数据去向**：每一条作业记录进入成长档案时间线 + 评分进入曲线图 + 同步至企微智能体可查询

### 5️⃣ 餐食与食育 🍱

**定位**：公共展示为主 + 个别孩子特别记录

| 环节 | 内容 |
|------|------|
| 买菜 | 拍买的菜 + 语音备注（"今天买了排骨、青菜"） |
| 制作 | 拍过程照片 1~9张 + 语音备注 |
| 成品 | 拍做好的饭菜 |
| 卫生记录 | 厨房消毒、食材新鲜度等打标 |
| 孩子们吃 | 合照/视频 + 语音备注总体情况 |
| 个别记录 | 表现突出的孩子关联档案 + 单独评语 |

> **数据去向**：主要进公共展示（欢迎页/招生素材），个别关联记录进成长档案。

### 6️⃣ 个人成长档案 🌟 ← 核心汇聚

不新建数据，而是**汇总所有模块记录，按时间线展示**：

```
签到记录 → 出勤统计 + 月历
作业记录 → 完成情况 + 评分 + 错题
拍照记录 → 精选照片展示
餐食记录 → 饮食情况
老师评语 → 文字记录
```

**展示内容**：
- **概览卡片**：当月出勤、平均评分、本周评语
- **成长时间线**：按日期排列（签到不显示，太频繁），有信息量的节点才展示
- **进步曲线**：
  - 学业评分趋势（折线图，近4周）
  - 各科对比（柱状图）
  - 出勤日历（绿/红标记）

### 7️⃣ 可视化总结报告 🎬

| 类型 | 频率 | 形式 | 制作方式 |
|------|------|------|----------|
| **月度报告** | 每月1次 | **H5动态页面**（有动画的翻页式） | 脚本半自动生成 |
| **期末短片** | 每学期1次 | **视频（MP4）** | **剪映**制作 |

**月度H5报告**：
- 照片+数据+评语的翻页式页面
- 各板块有淡入动画，折线图动态绘制
- 老师预览后分享给家长
- 先做简单版，后续深化

**期末短片**：
- 自然记录风格，不刻意叙事
- 内容包括：学习、玩耍、和小朋友接触
- 尽量记录好的方面
- **老师寄语（可选）**：老师录一段语音，最加分
- 工具：老师用**剪映**制作，系统负责整理素材+文案初稿

### 8️⃣ 家校沟通（企业微信客户群）💬

- **老师端**：企业微信
- **家长端**：普通微信（完全无感，不用装任何东西）
- **每个孩子一个专属群**（家长+老师+管理员）
- 群聊记录云端自动存档

**智能体（一期轻量版）**：
- 只做**数据查询员**，不扮演人、不编造
- 能回答：作业完成度、餐食记录、签到时间、最近照片、老师评语
- 不能回答：请假、投诉、纠纷 → 转真人老师
- 回复全部基于数据库真实数据，用固定模板输出
- 不说话时不存在群里

> **原则**：一个笨的智能体比没有更可怕，宁可不说也不乱说。

### 9️⃣ 收费管理 💰（与招生合并为对外展示模块）

**只记账，不收钱**（保持家长微信转账的现状）：

| 功能 | 说明 |
|------|------|
| 记一笔 | 哪个学生、什么费用、多少钱、交没交 |
| 缴费总览 | 谁交了谁没交，一目了然 |
| 催缴提醒 | 未交自动提醒老师 |

**对外展示**：展示收费标准（托管费/餐费/材料费），透明公开

### 🔟 招生营销 📢

- 精选素材库（老师平时标记"精选"的照片自动收集）
- 展示在欢迎页的"精彩瞬间"
- 收费标准展示
- 转介绍、意向客户跟进（待开发）

### 通知公告

- 放在**首页顶部循环滚动播放**
- 停课通知、调课、放假通知等

## 五、权限体系

| 角色 | 登录方式 | 能看什么 | 能做什么 |
|------|----------|----------|----------|
| 访客 | 无需登录 | 欢迎页、通知、收费标准、宣传素材 | 浏览 |
| 老师 | 管理密码 | **全部**学生全部数据 | 录入、修改、管理 |
| 家长 | 邀请码绑定微信 | 仅**自己孩子**的个人数据 | 查看，不能修改 |

## 六、技术选型

| 层 | 方案 |
|----|------|
| 前端 | **微信小程序**（唯一客户端） |
| 后端框架 | Python FastAPI |

| 云服务器 | 腾讯云/阿里云轻量服务器 |
