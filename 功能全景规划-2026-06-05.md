# 🏫 托班管理系统 · 功能全景规划

> **基于**：初版需求文档（2026-06-03）+ 与 Claude 的讨论（2026-06-05）

---

## 一、项目定位

| 项 | 内容 |
|------|------|
| 目标用户 | 托班老师（核心）、管理者 |
| 服务规模 | 约 20 名小学阶段儿童 |
| 核心理念 | 安全看护 + 作业辅导 + 成长可视化 |
| 使用原则 | ✅ **系统给老师用的**，家长端零操作或极低操作 |
| 沟通原则 | 一对一专属群（每个孩子一个群），替代大群刷屏 |

---

## 二、已确定的架构决策

### 整体架构

```
┌─────────────────────────────────────────────────────┐
│                   云服务器                           │
│  （腾讯云/阿里云 轻量服务器 ~1500元/年）              │
│                                                      │
│  ┌──────────┐  ┌────────────┐  ┌──────────────────┐ │
│  │ 后端API  │  │ PostgreSQL  │  │ 对象存储         │ │
│  │(FastAPI) │  │  数据库    │  │ (照片/视频素材)   │ │
│  └────┬─────┘  └────────────┘  └──────────────────┘ │
│       │ 企业微信回调接口                             │
│       │ 智能体调度服务                               │
└───────┼─────────────────────────────────────────────┘
        │ 双向同步
┌───────┼─────────────────────────────────────────────┐
│       │             本地电脑                         │
│  ┌────▼────┐  ┌───────────┐  ┌──────────────────┐  │
│  │本地服务 │  │ AI智能体  │  │ 视频渲染引擎     │  │
│  │(数据同步)│  │(Claude API)│  │ (FFmpeg + 模板)  │  │
│  └─────────┘  └───────────┘  └──────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 技术选型

| 层 | 方案 |
|----|------|
| **老师端界面** | 微信小程序（手机操作，随时拍照） |
| **后端框架** | Python FastAPI（或 Node.js） |
| **云端数据库** | PostgreSQL |
| **对象存储** | 云服务器本地磁盘 |
| **家校沟通** | 企业微信客户群（家长端普通微信，无感） |
| **AI 智能体** | Claude API + 自定义知识库 |
| **视频生成** | FFmpeg 模板化 + 本地渲染 |
| **数据备份** | 云端主存 + 本地同步副本 |

### 数据同步策略

| 数据方向 | 策略 |
|----------|------|
| 老师手机操作（拍照、签到） | ✅ 直写云端，即时生效 |
| 本地 AI 处理（视频渲染、报告） | 从云端拉数据 → 本地处理后写回云端 |
| 本地备份 | 定时自动从云拉取全量增量备份 |
| 冲突处理 | 以云端时间戳为准 |

## 三、功能全景图

### 模块 1：学生档案管理 👶

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 报名注册 | 录入姓名、年级、家长联系方式、紧急联系人 | ★★★ |

| 健康信息 | 过敏史、既往病史、特殊注意事项 | ★★★ |

| 家长授权管理 | 接送人授权名单（可授权爷爷奶奶、亲戚） | ★★★ |

## 四、开发路线图

### 一期 MVP（目标：4-6 周上线）

**核心原则**：让日常运转先跑起来

```
第1-2周  基础架构 + 学生档案 + 签到签退
第3-4周  拍照记录 + 作业管理 + 老师日常操作
第5-6周  企业微信客户群对接 + 成长档案基础版
```

**一期功能清单**：

| 模块 | 功能 |
|------|------|
| 学生管理 | 报名注册、档案卡、健康信息、接送授权 |
| 签到签退 | 每日到/离打卡、出勤记录、接送人记录 |
| 拍照记录 | 随拍关联学生、照片分类 |
| 作业管理 | 作业录入、完成跟踪、作业拍照 |
| 成长档案 | 时间线基础版、老师评语 |
| 报告系统 | 图文版每日/每周总结（PDF） |
| 企业微信 | 客户群建群、自动存档、（可选）智能体回复 |
| 餐食 | 每日餐食拍录、简单食育记录 |

**一期智能体能力**（轻量版）：
- 群聊智能体：回答"作业完成度"、"吃了什么"等常见查询
- 报告辅助：半自动生成每日总结文案

### 二期（目标：2-3 个月）

- 完整成长档案（时间线可视化、里程碑）
- 可视化总结报告（月报/季报）
- 视频报告初版（模板化合成）
- 食育记录完善 + 反差点推送
- 积分奖励体系
- 请假管理
- 收费管理基础版

### 三期（目标：3-6 个月）

- AI 视频自动生成
- 招生营销工具（精选素材库、转介绍）
- 意向客户跟进
- 数据分析看板
- 本地→云端完整同步体系
- 多角色权限管理

## 五、智能体（AI 助手）详细规划

### 5.1 企业微信群智能体

```
家长发消息 → 企业微信回调 → 云服务器 → 判断意图
                                          │
                    ┌─────────────────────┤
                    │                     │
              需要查数据              闲聊/无需回复
                    │                     │
              调用智能体               保持静默
              (查数据库+生成回答)           │
                    │                   不做操作
              调用企微API回复
```

**意图识别规则**：
| 家长说 | 智能体行为 |
|--------|-----------|
| "作业做完了吗" / "今天功课怎么样" | 查作业数据 → 回复完成情况 |
| "今天吃了什么" / "吃得好不好" | 查餐食/食育数据 → 回复 |
| "今天表现怎么样" / "乖不乖" | 查评语/精神状态 → 回复 |
| "睡午觉了吗" | 查作息记录 → 回复 |
| "拍了照片吗" / "看看他" | 查最近照片 → 发照片 |
| "@老师 我孩子发烧了今晚请假" | 转真人老师，不自动回复 |
| "好的谢谢" / "收到" | 不处理 |

**回答风格**：有温度，像真人老师
> ❌ "张小小今日作业完成率100%"
> ✅ "小小今天作业全部做完了！语文写字很工整，数学口算错了2道，已经订正过了👍"

### 5.2 运营辅助智能体

| 任务 | 输入 | 输出 |
|------|------|------|
| 写评语 | 原始记录（做了啥、状态如何） | 一段有温度的学生评语 |
| 写总结 | 一周数据+照片 | 周报文案 |
| 写通知 | 事件描述（停水/放假/调课） | 得体的通知文本 |
| 生成食谱说明 | 菜名+做法 | 吸引人的美食短文 |

## 六、待定/待决策事项

| 问题 | 选项 | 建议 |
|------|------|------|
| 开发方式 | 小程序 vs H5 vs App | **微信小程序**（老师手机操作最优解） |
| 后端框架 | FastAPI vs Node.js | **FastAPI**（Python 生态适合 AI 集成） |
| 视频生成方案 | 模板化 vs AI 生成 | 一期**模板化**→三期**AI生成** |
| 企业微信认证 | 个体工商户营业执照 | 需要，建议提前办理 |
| 智能体模型 | Claude API vs 国产 | **Claude API**（中文语境好，质量稳定） |
| 本地→云端同步 | 自建 rsync vs 第三方工具 | 建议**Syncthing**双向同步 |
